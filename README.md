# Информационная система для ЖК

Проект разработан в рамках хакатона. Это комплексное решение для управления информационными экранами в холлах, лифтах и на ресепшн жилых комплексов. Система позволяет автоматизировать вывод актуальной информации (новости, наличие парковочных мест) и оперативно оповещать жильцов в случае чрезвычайных ситуаций.

## Основные возможности

1.  **Панель управления УК:** Инструмент для настройки экранов, управления шаблонами виджетов и активации режима ЧС.
2.  **Дисплей клиента:** Адаптивное веб-приложение для отображения контента в портретном режиме (9:16).
3.  **Интеграция с Ujin API:** Автоматический сбор данных о новостях, свободных парковочных местах и кладовых.
4.  **Режим ЧС:** Мгновенное перекрытие любого контента важным уведомлением через WebSocket-соединение.
5.  **Симулятор экрана:** Консольное приложение для проверки работы API и WebSocket-событий.

## Структура проекта

Проект построен на микросервисной архитектуре с использованием Docker:

*   **`/backend`**: Сердце системы на **FastAPI**.
    *   `app/api`: API
    *   `app/application`: Бизнес-логика и сценарии использования (Use Cases).
    *   `app/domain`: Доменные модели (Entity) и интерфейсы.
    *   `app/infrastructure`: Реализация репозиториев (SQLAlchemy), провайдеров (Ujin API) и WebSocket-менеджер.
    *   `alembic`: Миграции базы данных (PostgreSQL).
*   **`/frontend`**: Одностраничное приложение (SPA) на **React + Vite**.
    *   Содержит два основных режима: `AdminPanel` (управление) и `DisplayPlayer` (плеер для экранов).
*   **`/simulator`**: Python-скрипт, имитирующий работу физического дисплея. Подключается по WebSocket и выводит состояние слотов в консоль.
*   **`Caddyfile`**: Конфигурация обратного прокси-сервера Caddy для маршрутизации трафика между фронтендом, бэкендом и WebSocket-сервером.

## Технологический стек

*   **Backend:** Python 3.12, FastAPI, SQLAlchemy (Async), PostgreSQL, Alembic, Pydantic v2.
*   **Frontend:** React 19, TypeScript, Vite, Tailwind CSS, Lucide React.
*   **Infrastructure:** Caddy (Reverse Proxy), Docker, Docker Compose.
*   **Real-time:** WebSockets.

## Запуск проекта

### 1. Подготовка окружения
Создайте файл `.env` в корневой директории проекта и заполните его данными (пример всех необходимых переменных приведен в файле `.env.example`)

### 2. Запуск через Docker Compose
Выполните команду для сборки и запуска всех контейнеров:

```bash
docker compose up --build
```

После успешного запуска система автоматически:
1. Поднимет базу данных PostgreSQL.
2. Применит миграции Alembic.
3. Запустит бэкенд, фронтенд и прокси-сервер.

## Ссылки и использование

После запуска проект доступен по следующим адресам:

*   **Панель управления (Admin):** [http://localhost/](http://localhost/)
*   **Список шаблонов/виджетов:** [http://localhost/templates](http://localhost/templates)
*   **Плеер экрана:** [http://localhost/player/lobby-1](http://localhost/player/lobby-1) (где `lobby-1` - slug созданного экрана)
*   **Документация API (Swagger):** [http://localhost/docs](http://localhost/docs)
*   **PGAdmin (БД):** [http://localhost:5050](http://localhost:5050)

### Быстрый старт:
1. Зайдите в **Swagger** и создайте новый экран через `POST /api/v1/screens/` (укажите `slug: lobby-1`).
2. Перейдите в **Панель управления**, найдите созданный экран и назначьте шаблоны в свободные слоты.
3. Откройте **Display Player** и проверьте отображение данных.
4. Попробуйте активировать **Режим ЧС** в админ-панели - экран мгновенно обновится без перезагрузки страницы.
